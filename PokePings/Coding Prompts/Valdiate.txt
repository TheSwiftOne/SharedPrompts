## Validation Quality Criteria (MANDATORY)

The implementation MUST satisfy:

- **Correctness**
  - matches design contracts exactly
- **Completeness**
  - all milestone requirements implemented
- **Robustness**
  - handles edge cases and failures
- **Consistency**
  - no regression or broken interfaces
- **Determinism**
  - behavior is predictable and testable

## Validation Scope

You MUST validate:

### 1. Contract Adherence

Verify:
- all interfaces match specs
- all schemas are respected
- no missing fields or mismatched types

### 2. Behavior Validation

Check:
- expected flows work
- invalid inputs are handled
- failure paths behave correctly

---

### 3. Test Coverage

Evaluate:
- are tests present?
- do tests map to design?
- do tests cover:
  - success cases
  - edge cases
  - failure modes

---

### 4. Regression Check

Ensure:
- no existing functionality is broken
- no unintended side effects introduced

---

### 5. System Integration

Validate integration with:
- store adapters
- subprocess model

- runtime config

## Evaluation Method (REQUIRED)

Use a structured evaluation loop:

### Phase 1 — Inspect
- review implementation
- identify mismatches

### Phase 2 — Critique
For each issue:

- what failed
- why it failed
- severity (blocking / non-blocking)

### Phase 3 — Decide

If ANY blocking issue exists:
→ milestone is NOT complete

If no blocking issues:
→ milestone is VALID

All issues MUST be categorized:

- **BLOCKING**

- **NON-BLOCKING**
  - improvement or optimization
  - does not affect correctness
